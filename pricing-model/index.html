Pebble Economics

What will Pebble cost to run?

The /brain direction turns Pebble into a data-rich engine — richer family context means more tokens on every call. This models the Claude API cost per family, weighs Sonnet against Opus, and shows where pricing needs to land.
The question

One subscription, one whole family.

Every family — the parents, the household, the extended ring (Tiers 1–3) — shares a single subscription. So the math is simple: what does one family cost to serve, and what should one family pay? Today PebblePath charges $1.99/month. That number was set when Pebble was light. A data-rich Pebble changes it.

A light Pebble answered the occasional question. The /brain Pebble runs a daily brief, ingests school emails and PDFs, synthesizes nightly notes, and reasons over a deep family memory on every call. Each of those calls now carries thousands of tokens of context. The cost per family moves from cents to dollars — and the calculator below shows by how much.

The recommendation

Where pricing needs to land.

Run Pebble on Sonnet, not Opus — and reprice to roughly $15/month.

Opus costs 4–5× Sonnet for the same work. At a realistic data-rich workload it lands near $25–75 per family per month — impossible to price for a consumer family app. Sonnet does the same job at $3–16, and with disciplined caching and usage caps the everyday family settles around $3–5. Today’s $1.99 cannot absorb that. The product has also changed category — from a milestone tracker to an AI chief of staff for the whole family — which carries the repricing. One subscription still covers everyone.

The three levers that decide the cost

1 · Prompt caching
The family-memory prefix is large and stable — cache it hard. Moving from 55% to 80% cache hit cuts input cost roughly in half. This is the single biggest lever.
2 · Usage caps
Caps convert an unbounded cost into a predictable one. Price each tier to the cost at its cap — an uncapped heavy family on Opus can cost $75/month.
3 · Model choice
Sonnet is the default. Reserve Opus, if used at all, for an occasional Pro-tier “deep reasoning” action — never the everyday path.

Why $14.99 holds: at 15% store fee, net revenue is $12.74/family. An engineered core workload — ~75–80% cache hit, usage capped near the light–typical band — runs about $3–5 in Sonnet API cost, i.e. 25–40% of net. That leaves room for Firebase, web search, support and a real margin. Annual billing at $149 improves cash and retention. Pushing to Opus, or leaving usage uncapped, breaks this.

How the cost is computed

Effective input = (context × cache factor) + variable input, where cache factor = miss share + hit share × 0.1. Cost = effective input × input rate + output × output rate + web-search cost. Net revenue = price − app-store fee.

This is a rough estimate, by design. It is meant to size the decision, not to be a billing forecast. It does not separately model cache-write premiums, prompt-caching TTL windows, retries, image/PDF parsing overhead, or non-AI infrastructure (Firebase, storage, Cloud Functions). Real cost per family will vary with how each family actually uses Pebble. Verify current Claude API pricing before any pricing decision is finalized.
